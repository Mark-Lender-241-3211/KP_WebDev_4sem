# Курсовой проект по дисциплине «Разработка веб-приложений»

Тема проекта: **«Разработка веб-приложения для автоматизации учета книг и читателей библиотеки»**.

Проект выполняется в рамках дисциплины «Разработка веб-приложений» по направлению подготовки 09.03.01 «Информатика и вычислительная техника».

## Цель проекта

Разработать веб-приложение для автоматизации учета книг и читателей библиотеки.

Приложение должно обеспечивать:

- аутентификацию и авторизацию пользователей;
- разграничение прав доступа в соответствии с ролью пользователя;
- CRUD-интерфейс для основных сущностей;
- обработку и агрегирование данных;
- загрузку и выгрузку файлов;
- хранение и обработку не менее трех сущностей.

## Предметная область

Разрабатываемое приложение предназначено для автоматизации базовых процессов библиотеки:

- учет книжного фонда;
- учет читателей;
- регистрация выдачи и возврата книг;
- поиск и фильтрация книг;
- формирование статистики по фонду и выдачам.

## Роли пользователей

В приложении планируется реализовать две основные роли:

1. **Библиотекарь**
   - управление книгами;
   - управление читателями;
   - оформление выдачи и возврата книг;
   - просмотр статистики;
   - загрузка и выгрузка файлов.

2. **Читатель**
   - просмотр каталога книг;
   - поиск и фильтрация книг;
   - просмотр информации о своих выдачах.

## Основные сущности

Планируемые сущности приложения:

- `Book` — книга;
- `Reader` — читатель;
- `Loan` — запись о выдаче книги;
- `Author` — автор;
- `Category` — категория книги;
- `User` — пользователь системы.

Минимальное требование о наличии трех сущностей выполняется за счет сущностей `Book`, `Reader` и `Loan`.

## Технологический стек

Планируемый стек разработки:

- Python;
- Django;
- SQLite;
- HTML;
- CSS;
- JavaScript.

Django выбран в качестве основного фреймворка, так как он содержит встроенные средства для работы с пользователями, группами, правами доступа, базой данных, формами и файлами. Это позволяет реализовать обязательные требования курсового проекта без подключения большого количества дополнительных библиотек.
